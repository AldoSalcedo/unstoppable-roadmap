# Agentes: Contexto & Estrategia de Ejecución

## Rol: Healthcare Software Engineer Builder

**Objetivo Principal**: Build a complete professional portfolio demonstrating expertise as a senior healthcare software engineer.

**Contexto Personal**:
- Background: Biology/QBP + Auditing experience
- Target Role: Senior Software Engineer, 100k+ MXN/month
- Languages: Spanish/English bilingual
- Unique Position: Domain expert in healthcare + technical depth

## Execution Strategy

### Phase 1: Discovery & Setup (Days 1-2)
- **Agent Task**: Research portfolio architecture patterns for senior engineers
- **Deliverable**: Next.js project scaffolding with TypeScript, Tailwind
- **Success**: Deployed skeleton on Vercel with working nav/hero

### Phase 2: Content Creation (Days 2-5)
- **Agent Task**: Write 5+ detailed project case studies with healthcare context
- **Deliverable**: Deployed projects section with metrics, code examples
- **Success**: Each project has: problem statement, solution, code examples, metrics, healthcare angle

### Phase 3: Thought Leadership (Days 3-6)
- **Agent Task**: Identify high-impact OSS projects; find "good first issues"
- **Deliverable**: 2-3 submitted PRs (ideally 1+ merged)
- **Success**: Merged contributions visible on GitHub profile and portfolio

### Phase 4: Marketing & Launch (Days 4-7)
- **Agent Task**: Optimize LinkedIn, craft resume, publish blog posts
- **Deliverable**: Live portfolio + updated LinkedIn + 3 blog posts + downloadable resume
- **Success**: Portfolio indexes in Google; LinkedIn profile updated; blog posts live

## Guardrails

### Quality First
- Every project case study must be real or realistically detailed
- Every metric must be plausible (don't make up unrealistic numbers)
- Every code example must be syntactically correct

### Healthcare Context Required
- Each project must clearly explain why it matters in healthcare
- Domain expertise (HIPAA, clinical workflows, data integrity) must be evident
- Compliance angle must be present (security, audit logging, etc.)

### Authenticity
- Portfolio must reflect real skills, not aspirational skills
- Case studies should be based on actual projects (past work or detailed examples)
- Blog posts should come from genuine experience, not copied content
