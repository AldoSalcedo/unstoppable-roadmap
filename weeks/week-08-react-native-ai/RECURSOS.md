```
Convention: ['entity', 'filter1', 'filter2', ...]
Examples:
- ['patients'] - all patients
- ['patients', patientId] - specific patient
- ['lab-results', patientId] - patient labs
- ['analysis', patientId, 'date'] - specific analysis
```

## Troubleshooting & FAQ

### Common Issues
1. **"Cannot find module" error**
   - Solution: `npm install` + restart simulator
2. **Simulator not starting**
   - Solution: `xcrun simctl erase all` (iOS) or reset AVD (Android)
3. **Build fails on EAS**
   - Solution: Check `app.json` syntax, validate env vars
4. **API calls failing offline**
   - Solution: Verify TanStack Query config has `networkMode: 'always'`
5. **AsyncStorage not persisting**
   - Solution: Ensure queries have `gcTime` set to non-zero

### Performance Tips
- Use `React.memo()` for expensive components
- Avoid creating new objects/arrays in render
- Use `useMemo` and `useCallback` strategically
- Profile with Flipper performance tab
- Test on real device (simulators lie about performance)

### Security Checklist
- [ ] API calls use HTTPS only
- [ ] Sensitive data not in Redux store (use query cache)
- [ ] AsyncStorage encrypted for PHI
- [ ] API keys in environment variables, not code
- [ ] Audit logging for data access
- [ ] Token refresh logic implemented

## Quick Start Commands

```bash
# Create new project
npx create-expo-app ClinicalApp

# Install dependencies
npm install expo-router expo-constants @tanstack/react-query axios

# Run on simulator
npx expo run:ios    # iOS
npx expo run:android # Android

# Build with EAS
npx eas build --platform ios --profile preview
npx eas build --platform android --profile preview

# Submit to stores
npx eas submit --platform ios
npx eas submit --platform android

# Run tests
npm run test

# Run e2e tests
npm run test:e2e

# Create deployment build
npm run build
```
