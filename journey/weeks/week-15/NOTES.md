## Day 1 — Research + Tracking System Setup

**Concepto**: Pipeline de aplicaciones = pipeline de ventas. Necesitas un sistema.

```markdown
| Company      | Role                    | Applied | Status    | Follow-up |
|--------------|-------------------------|---------|-----------|-----------|
| Zocdoc       | Sr. Software Engineer   | ✅      | Phone screen | 2026-04-20 |
| Oscar Health | React Native Engineer   | ✅      | Pending   | 2026-04-18 |
| One Medical  | Full-Stack Engineer     | ⏳      | Draft     | —         |
| Benchling    | Sr. Frontend Engineer   | ⏳      | Draft     | —         |
```

**Insight**: "Applied" sin seguimiento = desperdicio. Follow up siempre después de 7 días.

**Pregunta que surgió**: ¿Vale la pena aplicar por portal o buscar referido primero? → Referido aumenta probabilidad 5-10x. Intentar siempre LinkedIn primero.

## Day 2 — Resume Customization

**Concepto**: Un resume base → N versiones personalizadas. Highlight lo relevante por rol.

```
Healthcare companies → lead with QBP + auditing + HIPAA projects
AI companies → lead with Azure AI certification + clinical AI assistant project
Startup → lead with full-stack speed + React Native + shipped products
Enterprise → lead with clean architecture + testing + AWS certification
```

**Bullet formula (impacto)**:
```
❌ "Built authentication system"
✅ "Architected HIPAA-compliant auth with Auth.js + RBAC for 4 user roles,
    reducing unauthorized access incidents to 0 in 6-month production run"
```

**Pregunta que surgió**: ¿Pongo el sueldo objetivo en el resume? → Nunca en resume. Solo si piden en aplicación.

